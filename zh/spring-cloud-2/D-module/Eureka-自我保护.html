# 怎么禁止自我保护（一般生产环境中不会禁止自我保护）

# 注册中心 eureakeServer 端 7001

  1. 出厂默认，自我保护机制是开启的
eureka:
  server:
    # 关闭自我保护机制，保证不可用服务被及时清除
    enable-self-preservation: true
    # 时间间隔
    eviction-interval-timer-in-ms: 2000

  1. 使用 eureka.server.enable-self-preservation = false 可以禁用自我保护模式

  2. 关闭效果

  3. 在eurekaServer端7001处设置关闭自我保护机制

# 生产者客户端 eureakeClient 端8001

  1. 默认
eureka:
  instance:
    # Eureka 客户端向服务端发送心跳的时间间隔,单位为秒(默认是30秒)
    lease-renewal-interval-in-seconds: 30
    # Eureka 服务端在收到最后一次心跳后等待时间上限,单位为秒(默认是90秒)，超时将剔除服务
    lease-expiration-duration-in-seconds: 90
  1. 配置

  1. 测试
  • 7001和8001都配置完成
  • 先启动7001再启动8001
  • 先关闭8001
    • 马上被删除了
